# AAAI 投稿 Desk Check（格式与合规预检）

> 用途：在完整审稿前记录可观察到的投稿风险。每个结论均须标明其性质：**事实**、**分析**、**未核验**或**无法判定**。投稿缺陷不会停止后续审稿；仅输入不可读才停止流程。

## 1. 检查元数据

- 论文：`[填写：文件名或绝对路径]`
- 输入类型：`[填写：PDF / TeX / TeX 与编译 PDF]`
- 检查日期：`[填写：YYYY-MM-DD]`
- 网络状态：`[填写：可用 / 不可用 / 用户禁止联网]`
- 可读性：`[填写：可读 / 不可读 / 加密 / 正文无法解析]`
- 总体 desk 风险：`[填写：低 / 中 / 高 / 无法判定]`

## 2. 结论状态说明

| 状态 | 使用条件 |
|---|---|
| 通过 | 有直接证据表明该项满足已知检查条件。 |
| 警告 | 存在可观察风险，但不足以作出违规结论。 |
| 失败 | 有直接证据表明该项不满足可核验条件。 |
| 未核验 | 需要外部核验但网络不可用、被禁止或尚未完成。 |
| 无法判定 | 仅凭当前稿件或可用信息不能可靠判断。 |
